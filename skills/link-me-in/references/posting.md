# LinkedIn Posting

## Use this reference when

Use this reference for drafting, revising, reviewing, or publishing a LinkedIn feed post. For live posting, pair this with `references/official-posting-api.md`.

## Required inputs

Collect only what is needed:
- topic or announcement
- intended audience
- one clear takeaway
- supporting facts, links, names, and dates
- preferred tone: thoughtful, celebratory, practical, opinionated, or recruiting
- publish intent: draft only or publish after approval
- whether an official API client and OAuth setup already exist

## Drafting workflow

1. Identify the post type:
   announcement, lesson learned, milestone, hiring, event, recap, opinion, or resource share.
2. Extract the strongest hook from the source material.
3. Write a concise opening that earns a second line read.
4. Build 1 to 3 short body paragraphs with specific details.
5. End with one of:
   reflection, invitation to connect, call for discussion, or next step.
6. Add only relevant hashtags if they help discovery.

## Style guidance

- Prefer short paragraphs and clean line breaks.
- Lead with a concrete point, not a generic greeting.
- Use first person unless the user asks for a company voice.
- Keep the claim density high and the fluff low.
- Avoid overusing emojis, hashtags, and vague inspiration language.
- Mention other people or companies only when the user supplied or approved them.

## Review checklist

Check before publishing:
- facts are correct
- names and titles are spelled correctly
- links are valid
- confidential information is removed
- tone matches the user's brand
- call to action is optional and not forced

## Publish handoff

If direct posting is possible, show the final text and ask for explicit publish confirmation first.

If direct posting is not possible, return:
- final post text
- optional alternate hook
- suggested hashtags
- any manual posting steps needed
